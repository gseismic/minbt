# 常见金融交易策略概览

本文档整理常见金融交易策略的分类、核心假设、典型信号、适用场景与回测注意事项，供后续在 minbt 中设计策略示例、测试用例和风险检查时参考。

本文不构成投资建议。交易策略只有在清晰定义数据、成本、滑点、风控、容量和执行约束后才具备可评估性；历史回测表现也不能保证未来收益。

## 1. 长期配置与再平衡

长期配置强调先确定资产类别权重，例如股票、债券、现金、商品、数字资产等，再按固定周期或偏离阈值再平衡。其核心不是预测明天涨跌，而是控制组合暴露，让风险水平与投资期限、风险承受能力保持一致。

典型做法：

- 固定比例配置，例如 60% 风险资产、40% 防御资产。
- 固定周期再平衡，例如每月、每季度或每半年。
- 阈值再平衡，例如某资产权重偏离目标 5 个百分点后调仓。
- 风险预算配置，例如让每类资产贡献相近波动风险。

回测要点：

- 明确再平衡日和调仓价格。
- 计入交易费用和税费假设。
- 检查单一资产或单一行业集中度。
- 使用最大回撤、波动率、Calmar、夏普等指标评估风险收益，而不是只看最终收益。

主要风险：

- 极端市场中相关性可能上升，分散效果下降。
- 低频调仓可能错过快速风险变化。
- 过度再平衡会增加成本。

## 2. 定投与分批建仓

定投是在固定时间投入固定金额，不试图精准择时。SEC Investor.gov 将 dollar-cost averaging 描述为按固定金额、固定间隔持续投入，以降低一次性择时的影响。

典型做法：

- 固定日历频率买入，例如每周或每月。
- 对回撤做条件加仓，例如价格低于均线后增加投入。
- 对估值或波动做动态权重，但仍保持规则化执行。

回测要点：

- 区分策略收益与新增本金贡献。
- 净值曲线应同时展示总资产和单位净值。
- 对现金余额、手续费和小额订单精度做明确处理。

主要风险：

- 在长期下跌资产上持续投入会扩大绝对亏损。
- 定投降低择时风险，但不消除标的风险。
- 加杠杆定投会显著改变风险属性。

## 3. 趋势跟踪

趋势跟踪认为价格上涨或下跌趋势会在一段时间内延续。它通常不预测基本价值，而是用价格本身确认方向，在趋势形成时跟随，在趋势反转或波动异常时退出。

常见信号：

- 均线交叉：短均线上穿长均线做多，下穿做空或空仓。
- 通道突破：价格突破过去 N 日高点做多，跌破 N 日低点做空。
- 波动率过滤：只在波动率适中或趋势强度足够时交易。
- ATR 止损：用真实波幅设置动态止损距离。

适用场景：

- 趋势性强、流动性较好的市场。
- 多资产组合，因为单一资产可能长期震荡。
- 可接受低胜率但高盈亏比的资金管理方式。

回测要点：

- 避免用未来数据计算均线或突破阈值。
- 处理频繁假突破带来的手续费和滑点。
- 测试不同市场状态：趋势、震荡、暴跌、急涨。

主要风险：

- 震荡市中反复止损。
- 趋势末端追入导致回撤。
- 参数过拟合。

## 4. 动量与强弱轮动

动量策略基于相对收益排序，买入近期表现较强的资产，卖出或回避表现较弱的资产。它和趋势跟踪相似，但更强调横截面比较：不是只问某资产是否上涨，而是问它在候选资产中是否足够强。

常见信号：

- 过去 N 日收益率排名。
- 风险调整后收益排名，例如收益率除以波动率。
- 多周期合成动量，例如 1 月、3 月、6 月动量加权。
- 成交额或流动性过滤。

回测要点：

- 同一时间点只使用当时可见的全市场数据。
- 处理退市、停牌、上市时间不足等幸存者偏差。
- 明确每次轮动买入数量、权重上限和再平衡频率。

主要风险：

- 动量崩溃：市场快速反转时强者补跌、弱者反弹。
- 拥挤交易导致滑点上升。
- 高换手策略对成本非常敏感。

## 5. 均值回归

均值回归认为价格短期偏离均衡后会回到均值附近。该类策略常用于震荡市场、价差交易、短周期统计信号和流动性充足的标的。

常见信号：

- z-score：价格或价差偏离滚动均值若干标准差后反向交易。
- 布林带：价格触及上轨做空或减仓，触及下轨做多或加仓。
- RSI：超买超卖区间反向交易。
- 成交量冲击回归：异常放量拉升或下跌后等待回落。

回测要点：

- 均值和标准差必须滚动计算，不能使用全样本。
- 设置最大持仓时间，避免偏离持续扩大。
- 使用止损或仓位衰减，防止小盈利策略被单次趋势行情击穿。

主要风险：

- “均值”本身改变，例如基本面或制度变化。
- 下跌趋势中持续抄底。
- 波动聚集使历史标准差低估当前风险。

## 6. 配对交易与统计套利

配对交易选择两个经济关系较强的资产，构造价差或比值。当价差偏离历史区间时做多低估腿、做空高估腿，等待价差收敛。统计套利可扩展到多个资产和多个因子残差。

常见做法：

- 相关性筛选：先寻找历史相关性稳定的资产对。
- 协整检验：关注长期均衡关系，而不仅是短期相关。
- 回归残差交易：用一个资产或篮子解释另一个资产，交易残差偏离。
- beta 中性或美元中性：降低方向性市场暴露。

回测要点：

- 两条腿应同步成交，否则会产生腿风险。
- 价差阈值、止损、强制平仓规则必须明确。
- 回测中要记录总敞口、净敞口、保证金占用和单腿亏损。

主要风险：

- 相关关系失效。
- 价差长期不收敛。
- 借券、资金费率、交易限制和滑点侵蚀收益。

## 7. 套利

套利试图利用同一资产或高度相关资产在不同市场、不同期限、不同形式之间的价格错配。理论上套利风险较低，但真实执行通常依赖速度、资金、交易所规则和结算机制。

常见类型：

- 跨市场套利：同一资产在不同交易所价格不同。
- 期现套利：现货与期货或永续合约价格偏离合理基差。
- 三角套利：三种货币或交易对之间汇率不一致。
- ETF 申赎套利：ETF 价格与篮子资产净值偏离。
- 可转债套利：可转债、正股、期权特征之间定价偏离。

回测要点：

- 必须模拟双边报价、盘口深度、成交概率和转账/结算时间。
- 对手续费、资金费率、借贷成本、提现限制做完整建模。
- 对交易所宕机、单边成交、风控冻结等异常情形做压力测试。

主要风险：

- 看似无风险的价差无法成交。
- 单边成交后价格快速反向。
- 资金无法及时跨市场移动。

## 8. 做市与剥头皮

做市策略通过同时提供买卖报价赚取价差，并承担短期库存风险。剥头皮则追求非常短周期的小利润，可能是主动吃单，也可能是被动挂单。

CFTC 期货术语表中对 scalper 的解释强调其快速买卖、持仓时间短，并试图捕捉买卖价差、提供流动性。

常见信号与控制：

- 报价价差：根据波动率和盘口深度设置买卖价。
- 库存控制：多头库存过高时降低买价或提高卖出意愿。
- 订单簿失衡：盘口买卖量变化驱动短期方向判断。
- 延迟与撤单控制：防止过时报价被逆向选择。

回测要点：

- 需要盘口、逐笔成交、队列位置和限价单撮合。
- 只用 close 价格回测做市会严重高估收益或低估风险。
- 应评估成交率、逆向选择损失、库存回撤和手续费返佣。

主要风险：

- 极端行情中库存快速亏损。
- 技术延迟导致被动成交质量恶化。
- 手续费结构或交易所规则变化使收益消失。

## 9. 事件驱动

事件驱动策略围绕财报、并购、分红、回购、监管公告、宏观数据发布、链上事件等信息冲击进行交易。核心是判断事件影响是否已被价格充分反映。

常见类型：

- 财报后漂移：盈利超预期后价格继续反应。
- 并购套利：交易收购价与市场价差。
- 公告反转：利空或利好过度反应后回归。
- 宏观事件交易：利率决议、CPI、就业数据发布前后交易。
- 加密事件：升级、解锁、空投、交易所上线或下线。

回测要点：

- 使用事件发布时间，而不是事后整理日期。
- 避免把未来公告结果提前写入特征。
- 明确事件窗口，例如 T-1 到 T+5。
- 单独统计事件胜率、平均收益、尾部亏损和样本数量。

主要风险：

- 事件解释存在主观性。
- 样本稀少，统计显著性弱。
- 公告时间、停牌、流动性和跳空难以准确模拟。

## 10. 基本面与因子策略

基本面策略通过估值、盈利能力、财务质量、成长性、股东回报等指标选择资产。因子策略将这些可量化特征系统化，用排序、分组或优化方式构造组合。

常见因子：

- 价值：低估值资产相对高估值资产的潜在溢价。
- 动量：近期赢家继续表现较好的倾向。
- 质量：高盈利、低杠杆、现金流稳定的公司表现。
- 低波动：低风险资产可能有更好的风险调整收益。
- 规模：小市值资产在部分市场和周期中的溢价。
- 股息/回购：股东回报相关特征。

回测要点：

- 财务数据必须使用真实可得时间，不能用财报期末日期替代公告日期。
- 做行业和市值中性时，需要定义中性化方法。
- 因子应检验分层收益、换手率、IC、IR 和衰减速度。
- 组合构建要考虑权重上限、流动性、停牌和交易成本。

主要风险：

- 因子拥挤和长期失效。
- 财务数据重述导致历史数据污染。
- 多因子叠加时暴露重复或相互抵消。

## 11. 宏观与跨资产策略

宏观策略交易利率、通胀、增长、汇率、商品供需、央行政策等变量带来的跨资产重定价。它常见于期货、外汇、债券、商品和股指。

常见做法：

- 利率方向：根据政策周期交易债券或利率期货。
- 通胀交易：通过商品、通胀债或相关股票表达观点。
- 汇率趋势：基于利差、资本流动和政策差异。
- 股债轮动：根据增长和通胀状态切换风险资产和防御资产。
- 商品期限结构：根据现货供需和库存变化交易期货曲线。

回测要点：

- 宏观数据存在公布滞后和修订，必须使用当时版本。
- 跨资产交易要统一交易日历、汇率换算和合约换月规则。
- 需要评估低频信号与高频执行之间的匹配。

主要风险：

- 宏观变量与资产价格关系并不稳定。
- 政策冲击会快速改变定价逻辑。
- 杠杆和合约换月可能放大误差。

## 12. Carry 与期限结构策略

Carry 策略赚取持有某个头寸时自然产生的收益，例如利差、票息、股息、期货展期收益、永续合约资金费率等。Chicago Booth Review 对 carry 的介绍指出，它不只存在于外汇，也可扩展到多个资产类别。

常见类型：

- 外汇 carry：借入低利率货币，持有高利率货币。
- 债券 carry：持有收益率较高或曲线滚动收益较好的债券。
- 商品期限结构：在现货偏紧时利用 backwardation 或 contango 结构。
- 加密永续资金费率：在资金费率偏高或偏低时构造现货/合约或多空组合。
- 股指期货基差：交易期货价格相对现货指数的偏离。

回测要点：

- 收益不仅来自价格变化，还来自资金费率、票息、展期和借贷成本。
- 必须记录持有期间的现金流。
- 在压力期模拟 carry unwind，也就是拥挤头寸集中平仓。

主要风险：

- 低波动时期收益稳定，危机时期可能一次性回吐多年收益。
- 融资成本和保证金要求变化。
- 流动性下降导致无法平仓或展期。

## 13. 波动率与期权策略

期权策略交易方向、波动率、时间价值和尾部风险。期权是衍生品，其价值来自标的资产，SEC Investor.gov 的期权投资者公告也强调了期权术语和风险教育的重要性。

常见类型：

- 备兑看涨：持有现货并卖出看涨期权。
- 保护性看跌：持有现货并买入看跌期权对冲尾部风险。
- 跨式/宽跨式：押注大幅波动或波动率变化。
- 波动率套利：交易隐含波动率与实现波动率差异。
- 尾部保护：长期持有深度虚值保护性期权。

回测要点：

- 需要期权链、隐含波动率、到期日、行权价、无风险利率和分红。
- 不能只用标的价格近似期权收益。
- 应计算 delta、gamma、theta、vega 等风险暴露。

主要风险：

- 卖出期权可能承担非线性尾部亏损。
- 期权流动性和买卖价差对收益影响很大。
- 波动率曲面变化会让简单方向判断失效。

## 14. 机器学习与数据驱动策略

机器学习策略用模型从大量特征中预测收益、方向、波动率、成交概率或风险状态。它不是单独的收益来源，而是一种信号生成方法，可以服务于趋势、均值回归、事件或组合优化。

常见任务：

- 分类：预测下一周期上涨、下跌或震荡。
- 回归：预测未来收益率、波动率或成交量。
- 排序：对多标的预期收益打分。
- 状态识别：识别趋势市、震荡市、高波动市。
- 执行优化：预测滑点、成交概率和冲击成本。

回测要点：

- 严格时间切分训练集、验证集和测试集。
- 使用 walk-forward 或滚动训练，避免未来泄露。
- 报告样本外表现，而不是只看训练期。
- 将交易成本、换手和容量纳入目标函数。

主要风险：

- 过拟合和数据泄露。
- 特征在市场环境变化后失效。
- 模型复杂度掩盖了真实经济逻辑。

## 风险管理清单

无论策略类型如何，回测前都应显式定义以下约束：

- 交易标的：交易所、品种、交易时间、停牌或断流规则。
- 数据口径：价格字段、复权方式、成交量、资金费率、财务数据发布时间。
- 成交模型：市价、限价、滑点、盘口深度、部分成交、最小下单单位。
- 成本模型：手续费、返佣、借贷成本、资金费率、税费、换汇成本。
- 仓位管理：单笔风险、总杠杆、净敞口、行业敞口、相关性敞口。
- 退出规则：止损、止盈、时间止损、信号反转、强制减仓。
- 风险指标：最大回撤、波动率、VaR、尾部亏损、爆仓概率、保证金占用。
- 稳健性：参数敏感性、样本外测试、分市场状态测试、极端行情压力测试。

## 与 minbt 的落地优先级

minbt 当前强调 T+0、高频换仓、多空双向、市价撮合、保证金和多 portfolio。基于这些能力，后续策略样例可以按以下优先级推进：

1. 均线趋势策略：单标的多空，验证市价单、多空翻转和杠杆。
2. 布林带均值回归：单标的短周期，验证频繁换仓、手续费和止损需求。
3. 多标的动量轮动：验证 `date_key` 排序、多 symbol 行情和组合权益统计。
4. 配对价差策略：验证多腿持仓、净敞口和 portfolio 风险隔离。
5. 永续资金费率 carry：如果引入资金费率字段，可验证现金流、持仓收益和保证金。
6. 多 portfolio 策略组合：一个 portfolio 跑趋势，一个跑均值回归，验证分仓权益和风险隔离。

当前不宜优先用 minbt 做严肃做市、盘口套利或期权策略，因为框架尚未支持限价单、订单簿、部分成交和期权定价模型。可以先在文档或示例中标注为“研究方向”，等撮合和合约模型完善后再实现。
