# PLAN-014 修复目标规范化与 A 股 dt 边界

## 背景

再次 review 后确认两个问题：

1. `submit_market_order()` 会对用户显式传入的 A 股数量做整手规范化，导致 `qty=150` 静默成交 100。
2. Exchange 不传 `date_key` 时使用整数行号作为 `dt`，A 股市场把整数转换为 `1970-01-01`，导致缺少真实交易日期时仍可交易。

## 目标

1. 显式市价单不做数量规范化，非整手应按市场规则拒单。
2. 目标仓位接口继续允许按市场规则规范化目标订单数量。
3. A 股市场拒绝数字型 `dt`，要求真实日期或时间。
4. 增加回归测试覆盖上述行为。

## 非目标

1. 不改变 `SimpleMarket` 的行号时间语义。
2. 不实现交易日历。
3. 不实现更复杂的订单类型。

## 验证

1. 运行定向测试。
2. 运行全量 `pytest -q`。
3. 运行 `python -m compileall -q minbt tests examples`。
4. 运行 `git diff --check`。
